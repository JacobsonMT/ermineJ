Gene Ontology analysis for  microarray data

Overview

ErmineJ performs analyses of gene sets in expression microarray data. A typical goal is to determine whether particular biological pathways are "doing something interesting" in the data. The software is designed to be used by biologists with little or no informatics background. A command-line interface is available for users who wish to script the use of ermineJ.

Key features include:

  • Implementation of several methods for gene set analysis:
    • Over-representation analysis
    • A resampling-based method that uses gene scores
    • A resampling-based method that uses correlation between gene expression profiles .
  • Gene sets receive statistical scores (p-values), and multiple test correction is supported.
  • Support of the Gene Ontology terminology
  • User files use simple text formats.
  • Users can modify gene sets or create new ones.
  • The results can be visualized within the software.
  • It is simple to compare multiple analyses of the same data set with different settings.
  • Hyperlinks are provided to external sites to allow more efficient browsing of the results.

Download and installation

Requirements

ermineJ requires a Java 2 Runtime Environment (JRE) 1.4 or newer to run. You can download the platform-specific packages from SUN's website (http://java.sun.com/j2se/ ). Users of MS Windows NT/2000/XP can alternatively download the Windows installer with bundled JRE package below.

Software Files

Description File Size
Windows installer with bundled JRE ermineJ-setup-jre.exe ~30 MB
Windows installer without JRE ermineJ-setup.exe ~7 MB
Generic bundle ermineJ-2.0-bindist.tar.gz ~7 MB
Source code ermineJ-2.0-src.tar.gz < 1 MB

Annotation files

To use the software, you have to have microarray annotation files and a recent Gene Ontology XML file.

Go here to get annotation files.  Unzip the one(s) you need and put them somewhere you can find them.

Go here to get the XML file (something like http://www.godatabase.org/devdatabase/archive/latest/go_200409-termdb.xml.gz; the date portion of the name may have changed). Unzip it and put it somewhere you can locate it later.

Installation

On MS Windows NT/2000/XP, simply run the installer.

For other platforms, you should be able to use the "Generic bundle". There is a README file included that gives some instructions.

WebStart

You can run ermineJ from any operating system that supports Java (in theory) without running the installer by using java webstart.

This a good way to get the latest version, as we will keep the link updated with the most recent development build. On the other hand, this version should be considered less stable than the other releases. This is here partly for our own testing purposes.
